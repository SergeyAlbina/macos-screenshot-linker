# Вклад в проект
Язык: [English](CONTRIBUTING.md) | [Русский](CONTRIBUTING.ru.md)

## Разработка

1. Сделай форк репозитория
2. Создай feature-ветку
3. По возможности сохраняй POSIX-совместимость скриптов
4. Запусти проверки перед открытием PR

## Локальные проверки

```bash
shellcheck scripts/*.sh
```

## Pull Request

- Один логический набор изменений на один PR
- Обновляй документацию, если меняется поведение
- Для багфиксов добавляй шаги воспроизведения
