# Contributing
Language: [English](CONTRIBUTING.md) | [Русский](CONTRIBUTING.ru.md)

## Development

1. Fork the repo
2. Create a feature branch
3. Keep scripts POSIX-friendly where possible
4. Run checks before opening a PR

## Local checks

```bash
shellcheck scripts/*.sh
```

## Pull requests

- One logical change per PR
- Add/update docs for behavior changes
- Include repro steps for bug fixes
